식별자/키워드

키워드란?



자바 스크립트가 처음 만들어질 때 정해진, 특별한 의미가 있는 단어이다.
모든 브라우저에서 아래와 같은 키워드들을 지원하며, 이 키워드들은 식별자로 정의할 수 없다

키워드 종류

 
    var        변수를 선언한다.  
    function     함수를 선언한다.
    return       함수에 return 뒤에 있는 값을 반환하고 함수를 종료한다. 
    break       loop문이나 switch문을 종료하기 위해 사용.
    continue    loop문을 빠져나가 가장상단으로 가는데 사용.
    const       const는 상수(변하지 않는 값)을 위해 사용합니다.
    new         생성자 함수를 실행한다.
    else         if문 실행
    extends    클래스를 다른 클래스의 자식으로 만들기 위해 class 선언 또는 class 식에 사용됩니다.
    debugger   자바스크립트의 실행을 중지하고 (가능하다면)디버깅 function을 부른다.
    try         error를 핸들링하는 try catch문 실행
    default     사용자가 특별히 명령을 내리지 않으면 시스템이 자동으로 적용하는 미리 정해진 값이나 조건.
    delete      피연산자로 지정된 객체 프로퍼티, 배열원소, 변수의 삭제를 시도하는 단항연산자이다.
    while       catch절 다음에 finally 블록이 이어지는데, 앞서 try 블록에서 일어난 일에 관계없이 항상 실행이 보장되어야 할 뒷정리용 코드가 포함된다.
    if           if문 실행
    super       부모 오브젝트의 함수를 호출할 때 사용됩니다.
    instanceof   A instanceof B의 형식으로 사용하며, A가 B의 인스턴스가 맞는지 판단한다.
    yield       제너레이터 함수 (function* 또는  레거시 generator 함수)를 중지하거나 재개하는데 사용됩니다.
    this        for반복문 실행
    with        객체를 제어하는 명령어로, 특정 객체를 여러번 사용할 경우 생략할 수 있도록 해준다.
    switch      switch문 실행
    case        표현식은 CASE 키워드로 시작하고 END 키워드로 끝납니다. 중간에는 숫자 섹션 또는 '절'이 있습니다.
    in         키워드는 객체를 생성하는 방법 중, 생성자 함수(Constructor) 를 사용하여 객체를 만들 때 함께 쓰는 키워드이다.
    throw       서드를 호출한 곳으로 예외를 위임(떠넘김) 할 때 throws 키워드를 사용한다.
    typeof      변수의 데이터형을 알아내기 위해서 사용하는 예약어입니다.
    void        연산자는 주어진 표현식을 평가하고 undefined를 반환합니다


식별자란?

식별자는 자바스크립트에서 이름을 붙일 때 사용하는 단어이다. 식별자의 예로는 변수명과 함수명이 있다.

식별자 규칙

★ 최소한 하나의 문자로 구성 ★ 첫 번째 글자는 반드시 하나의 문자, _, & ★ 숫자로 시작 불가능
★ 공백 포함 불가능 ★ 첫 글자 다음에 오는 각 글자는 _, &, 숫자 ★ 자카스크립트 키워드 사용 불가능

식별자 관행

★ 모든 언어 사용가능 하지만 영어 사용이 관례 ★ 함축적이고 의미 있는 단어 사용 ★ 생성자 함수의 이름 대문자로 시작
★ 변수와 인스턴트, 함수, 메서드 이름은 소문자로 시작 ★ 식별자가 여러 단어로 이루어 지면 각 단어의 첫 글자는 대문자.

식별자 표기방법


카멜 표기법(CamelCase)
첫 문자를 소문자로 표기하고, 그 다음 문자부터는 대문자로 표기 합니다.
이것을 단봉낙타 표기법이라고도 불립니다.
가장 권장되는 표기법 입니다.

 
    EX) backgroundColor, typeName, iPhone

파스칼 표기법(PascalCase)
첫 문자를 대문자로 표기하고, 그 다음 문자도 대문자로 표기하는 표기법입니다. 이것을 쌍봉낙타 표기법이라고도 불립니다.
함수와 클래스명은 이 표기법을 권장한다고 합니다

 
    EX) class GameEngine{};, namespace BaseData{}

스네이크 표기법(snake_case)
한 문자마다 _(underscore)를 붙여 이어나가는 표기법을 말합니다.
변수명, 함수명, 데이터 타입, 네임스페이스명 등에 사용된다고 합니다.

 
    EX) String human_name;, void call_someone() {}
